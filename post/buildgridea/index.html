2020年，教你如何在Coding上白嫖一个Gridea博客

· 2020-08-27 · # Coding # Gridea

😮 都2020年了还不会整博客嘛？来，我教你！🙌

前言

适合刚刚入门，想写博客的朋友。
成本低：服务器白嫖，域名可以自己买，甚至不介意的话，用coding给你的域名也可以。
上手快：相对于hexo，gridea的部署真的可以说是傻瓜式了，基本上全部都是可视化操作。
开整开整

gridea

官网：https://gridea.dev/
Gridea 是一个静态博客写作客户端
我当前试用的版本是 v0.9.2

Coding

一站式 DevOps，提升研发效能全新 CODING 涵盖了软件开发从构想到交付的一切所需，使研发团队在
云端高效协同，实践敏捷开发与 DevOps，提升软件交付质量与速度。
我们就是把博客部署在这个平台上，相对于github page，国内访问速度应该会快很多。
官网：https://coding.net/

step1 在Coding上部署静态网站

在coding上注册账号，然后创建项目，选择项目模板，我选择得是DevOps 项目

项目新建之后，填写项目信息，这里比较重要得是项目名称

新建完我们进入项目页，此时我们需要以此点击以下操作来进行静态pages部署
首先点击左下角得项目设置

在功能开关里找到持续部署打开它

然后我们返回项目页，在持续部署中找到静态网站，进入之后点击立即发布静态网站（需要进行账号实名认证）

创建完静态网站后，点击立即部署按钮开始进行网站部署。
部署完成后就可以访问默认的coding page页面了

如果自己有域名，可以点击右上角的设置，进行自定义域名
域名解析需要用CNAME记录指向你的coding页面地址 xxxxxx.coding-pages.com

step2 设置Gridea

先去gridea官网下载对应平台的客户端

进入客户端之后点击远程

下面进行配置

  • 域名
    填Coding静态网站给我们的域名，比如如果你绑定好了你自己的域名，直接填写你的域名也可以
    域名之前的https://和http://，如果静态网站开启了强制https，那你就选https://

  • 仓库名称（一定要注意！注意！）
    这里一定要填公开仓库的名称
    点击代码仓库，选访问设置，然后点击开源访问地址

    现在得到的这个才是仓库名称

  • 分支
    这里我们默认是master

  • 仓库用户名
    项目首页-代码仓库-仓库设置-仓库名称，如图框选的就行用户名

  • 邮箱
    可以填也可以不填

  • 令牌用户名和令牌
    在coding，点击右上角头像，进入个人账户设置，点击访问令牌，然后新建令牌

    这里只需要勾选第一个 project:depot

    输入coding的账户密码就可生成令牌了，然后把令牌和令牌名填入gridea。

  • CNAME
    如果绑定了自己域名可以填写自己的域名地址

    点击右下角保存，然后检测远程连接，远程连接成功，gridea就算是设置好了
